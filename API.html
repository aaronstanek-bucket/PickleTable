PickleTable

This is the guide that describes how to use the PickleTable Python Module. The master copy of this module can be found at https://github.com/aaronstanek/PickleTable
I hope that you are able to find this software useful. --Aaron Stanek

About

PickleTable is a Python Module whose purpose is to guarantee that data is being saved in a desirable and easily usable format. In short, it provides an interface with Python's Pickle Module, and checks data format and datatype while saving. The files generated and read by this module will always have the extension .pickletable (this is to make it harder to load malformatted data). Each file is a collection of 2 dimensional tables. The width of each table must be fixed before any data can be written to the table. The entries in each table are constrined to the types: int, float, string, and boolean. The file as a whole, as well as each table can have a name and human readable description.

API

The module contains a class called PickleTable. The API for this module is built around specific methods in this class. Making function calls to things in the module, but outside the API, may lead to undefined or undesirable behavior.
Each table and PickleTable has a name and a readme. Both are strings and can be set arbitrarily by the user (they have no predetermined purpose). When sending data to a PickleTable you will have to keep track of the active table. You can only ever edit the active table. The active table is set by the addTable method. (The newest table is always the active one). So, after you create a new table, you cannot go back and edit a table that you created previously.
Listed below are the functions that comprise the PickleTable API. All are methods of a PickleTable object, so to use this module, you must first create a new PickleTable object, then you must use the methods listed below to modify/query the PickleTable object.

Saving and Loading
toFile(filename) This saves a PickleTable object to a file.
The filename specifies where to save the object. This argument must be an string.
The filename must end in ".pickletable"
fromFile(filename) This loads a PickleTable object from a file.
The filename specifies where to get the object. This argument must be an string.
The filename must end in ".pickletable"
Note that this operation will overwrite any information that is presently stored in the PickleTable.

PickleTable level operations
clear() This method will reset a PickleTable. The name, readme are set to the empty string, and all tables are dropped.
setName(name) This method will set the name for a file. (Note: this is not the filename, this is a piece of information stored in the file called "name")
The name argument must be a string.
setReadme(readme) This method will set the name for a file.
The readme argument must be a string.
getName() This returns the value of the name property of a PickleTable.
getReadme() This returns the value of the readme property of a PickleTable.
getTableCount() This returns the number of tables stored in a PickleTable.
addTable() This adds a new table to a PickleTable. This new table is then set as the active table.

Table level operations
setTableName(name) This will set the name of the active table.
The name argument must be a string.
setTableReadme(readme) This will set the readme of the active table.
The readme argument must be a string.
setTableWidth(width) This will set the number of columns in the active table.
It is only possible to use this function if the active table has zero rows. If the table has a nonzero number of rows then an exception will be thrown. You may use getTableRowCount to determine the number of rows in a table, or clear to empty everything.
The width argument must be a int.
getTableName(index) This will return the name of a particular table in a PickleTable.
The index argument specifies which table you would like to know about, and must be an int.
getTableReadme(index) This will return the readme of a particular table in a PickleTable.
The index argument specifies which table you would like to know about, and must be an int.
getTableWidth(index) This will return the width of a particular table in a PickleTable.
The index argument specifies which table you would like to know about, and must be an int.
getTableRowCount(index) This will return the number of rows in a particular table in a PickleTable.
The index argument specifies which table you would like to know about, and must be an int.

Row level operations
add(row) This will append a row of data to the active table. Row must be a list and must have the correct number of elements. The correct number of elements is determined by the active table's width, which must be set before any rows are added. If you try to add a row before the width has been set, then an exception will be thrown. The elements of the row argument may only be int, float, str, or bool. The type will be check before the information is added to the table.
get(tindex,rnum) This will return a row of data. tindex is the table index; it specifies from which table you would like information. rnum specifies which row in the table you would like.
Both the tindex and rnum arguments must be int.
